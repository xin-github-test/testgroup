1.pwd

  查看当前目录的所有内容信息（也可以显示文件的信息，比如大小等)
   
   可带参数用法：

   可带参数用法：

 more指令中有若干快捷键

 如：空格：向下翻一页     enter: 向下翻一行  q:代表立刻离开more,不在显示该文件的内容
      ctrl+F  ：向下滚动一屏    ctrl+B :返回上一屏    =：输出当前行号     :f ：输出文件名和当前行的行号	
12.less
用来分屏查看文件的内容，它的功能和more有些类似，但比more更强大，支持各种显示终端。less指令在显示文件的时候
并不是一次将整个文件都加载才显示，而是根据需要显示，对于大型文件具有较高的效率
 其中的快捷键

如：空格：向下翻一页   pagedown:向下翻一页  pageup:向上翻动一页    /字串：向下搜寻【字串】的功能；n:向下查  N:向上查
                   ？字串：向上搜寻【字串】的功能，n：向上查，N:向下查；  q:离开less这个程序
13.>和>>
 > :输出重定向（会将源文件内容覆盖）
>>:追加（不会覆盖原来文件的内容，而是追加到文件尾部）
基本语法：
  ls -l > 文件  ：列表的内容写入文件中（覆盖写）若文件不存在，先生成再覆盖
  ls -al >> 文件：列表的内容追加到文件的尾部
  cat 文件1 > 文件2  ：将文件1的内容覆盖到文件2
  echo "内容" >> 文件 ：将内容追加到文件尾部

三、组管理
1.ls -ahl
 查看文件所有者
groupadd 组名 ：添加组
useradd 成员：添加成员  （可带参数-g 组名）添加一个用户并将其加入指定的组
一般文件由谁创建的的，该文件的所有者就是谁，且文件的组也是和所有者的组一样，但是文件的所有者和组时可以改变的
2.chown
 修改文件的所有者
  chown 用户名 文件
还有一种用法：chown newowner:newgroup 文件名 ：同时改变文件的所有者和所在组
如果是目录则 加上-R 使得目录下的子文件和子目录一起递归改变
3.chgrp
 修改文件所在组
 chgrp 组名 文件名
 其他组的概念：除了文件的所有者和所在的用户外，系统的其他用户都是文件的其它组
4.改变用户所在组
 usermod -g 组名 用户名
 usermod -d 目录名 用户名  :改变用户登陆的初始目录
5.chmod 
  修改权限（修改文件或目录的权限）
第一种方式：+ - =变更权限
 u:所有者  g:所在组  o:其它人  a:所有人（a,g,o的总和）
 还可以用数字代表r,w,x，其中r为4，w为2，x为1，所以rw为6，rwx为7
 1) chmod u=rwx,g=r,o=x 文件目录名
 2）chmod o+w 文件目录名
 3）chmod a-x 文件目录名

 四、磁盘分区、挂载
1.查看系统的分区和挂载的情况：lsblk -f 或者lsblk

 挂载的经典案例：
需求：给linux系统增加一个新的硬盘，并且挂载到/home/newdisk
步骤：
 第一步：虚拟机添加硬盘
    直接在虚拟机的设置里添加一块硬盘就行，设置成功耦重启才能看到
 第二步：分区
  命令 ：fdisk /dev/sdb    进入新的硬盘目录
  接着输入m,在输入n添加一个新的分区，根据提示分区完成后，w保存并退出
 第三步：格式化
  分区完成后也不能存储东西，需要进行格式化
 命令：mkfs -t ext4 /dev/sdb1    将刚创建的分区格式化成ext4格式
 第四步：挂载
命令： mount /dev/sdb1 /home/newdisk
执行到此处已经挂载成功，但是重启之后，需要重新挂载，所以需要进行下一步，自动挂载，也就是永久挂载
 第五步：设置可以自动挂载
 命令 ：vim /etc/fstab,将你的分区挂载信息写上去保存就行，uuid可以直接替换成绝对路径
保存之后 使命令 mount -a 设置自动挂载就可以了

 如果需要解除挂载可以使用命令 ：umount /dev/sdb1（注意：解除挂载的时候不应该在其挂载的目录里）
2.df -h或df -lh
磁盘情况查询
 基本语法：
 df -h
3.du -h
查看指定目录的磁盘的占用情况
 基本语法：
  du -h /目录  （默认为当前目录）
 常用选项：
  -s :指定目录占用大小汇总
  -h :带计量单位
  -a :含文件
  --max-depth=1 ：子目录的深度 （注意：开头俩个杠）
 -c :列出明细的同时，增加汇总值
4.常用命令：
  1）统计/home 文件夹下的文件的个数：
   命令：ls -l /home | grep "^-" | wc -l    (^代表定位符，后面的-代表以-开头的文件，wc为统计个数）
  2）统计/home文件夹下目录的个数
   命令：ls -l /home | grep "^d" | wc -l   (d代表该文件是目录)
  3）统计/home文件夹下文件的个数，包括子文件夹的
   命令 :ls -lR /home | grep "^-" | wc -l  (R代表递归，这里为大写的)
  4)统计/home文件夹下目录的个数，包括子文件夹的
   命令：ls -lR /home | grep "^d" | wc -l
  5）以树状显示目录结构
   命令：tree   (如果没有此指令可以用yum install tree 安装指令)

五、进程管理：
1.ps
 可以用来查看目前系统中，有哪些正在执行进程，以及它们执行的状况，可以不加任何参数
一般使用的参数是 
                           -aux  其中a代表显示当前终端的所有进程信息
                                           u代表以用户的格式显示进程信息
                                           x代表显示后台程序运行的参数
                           -ef  可以看到进程的父进程，就是ppid的那一列

                    TTY:终端机号

  基本语法：
  kill [选项] 进程号  ：通过进程号杀死进程
  killall 进程名称     ：通过进程名杀死进程，也支持通配符，这在系统因负载过大时
而变得很慢时很有用
  常用选项：
 -9 ：表示强迫进程立即停止
 3.查看进程数 pstree
pstree [选项] ：更加直观的观看进程信息
常用选项：
   -p :显示进程的pid
   -u：显示进程的所属用户

1.service 服务名 [start | stop | restart | reload | status]
在centos7.0 后就不是用service  而是 systemctl
注意：

      2.这种方式只是临时生效，重启系统后还是回归以前对服务的设置

  方式2：/etc/init.d/服务名称

  1）查看服务 chkconfig --list | grep xxx

七、进程监控
1.netstat [选项]
  监控网络状态的指令
  选项说明：
    -an :按一定顺序排列输出
    -p  :显示哪个进程在调用
  一般使用：netstat -anp
2.top 
 top 指令和ps命令很相似，都是用来显示正在执行的进程，但是top在执行一段时间可以更新正在运行的进程
  所以说是动态的
 基本语法：
 top [选项]
  选项说明： 
  -d 秒数  ：指定top命令每隔几秒更新，默认是3秒，在top命令的交互模式当中可以执行的命令
  -i    :使top不显示任何闲置或者僵死的进程
  -p  :通过指定监控进程的ID来仅仅监控某个进程的状态
交互操作说明：
  p :以CPU使用频率排序。默认为该项
  M:以内存的使用频率排序
  N:以PID排序
  q:退出top
  u:指定查看的用户
  k:杀死指定的进程
